Hexo theme construction(1)

六月 17 2016

在经历了上会的小挫折，让我重新对Hexo的模版结构进行了研究和学习。下面就进行一个小小的分享。

Hexo采用了类似于Django的MVC模型，每个模版都有独立的CSS配置，所以你可以随心所欲的对每个模版进行设置而不用担心会影响到其他的部分，除非你在模版引用的时候引用了相同的文件。

然后是模版类型，有多种，例如：

  • ejs
  • jade
  • swig
  • haml

前两种是比较流行的，第一种是类似于html的标签式模版，是我比较喜欢的，于是在接下来的示例中，都将选用ejs。

Hexo的结构十分明显：

_config.yml

这是一个配置文件，你可以将一些基础的配置文件写入，比如设置一些快捷URL，比如设置Github主页链接，或者是作者信息，甚至是网站图标、首页的图片，都可以在这里进行快捷设置。

languages

可以设置网站的语言，相当于http协议里面的zh-cn吧，可以查看官方网站查看更多信息。

source

这个部分通常用于放置静态文件，这部分将在接下来的文章中进行详解。

layout

这里就是模版放置的位置，可以对每个模版的细节进行配置。

如果有对Hexo隐藏的文件，可以在改文件名前加“_”。

这部分的结构主要为一下几部分构成

index

主页的模版。

layout

文章模版。

archive

相当于目录的作用。

page

这部分则是我的博客里面的POST。

post

这部分是设定post的内容。

scripts

这部分则是js放置的位置了。
