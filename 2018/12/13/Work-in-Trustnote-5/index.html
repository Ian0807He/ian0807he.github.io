DAG Is Future

The biggest advantage of DAG is tps. You can compose a joint whenever you want. The only thing you need to do next is waiting someone to compose another joint which chose your joint as a parent joint. So the tps of the DAG can be infinity. The only limit is the band width of network and the performance of computer.

So it’s not very easy to control the whole chain by using 51% computing power attack. You cannot create a double spend and only admit the bad one by permanent divergence. Once a unit is included by other unit, it will stay in the graph.

DAG Is Good But Not Perfect

There is still things that DAG cannot process perfectly. In blockchain like Bitcoin, you don’t need to worried too much about the transations inside the block which will be verified by the next blocks. Which means you don’t need to verified a whole UTXO. But in DAG like byteball, you need the full details about the history or you can’t verify anything. Which also means you can’t process units with async in most situation. Which is a very bad for catchup.

And because there is latency between nodes, double spend unit can will stay in the graph. So, when a new node wants to verify a transaction, the node need the full history of the UTXO to verify the double spend. Therefore, SPV in pure DAG is impossible, I think.
